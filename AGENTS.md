# Agent Instructions: Meditrendy Pro Child Theme

This repository contains the Meditrendy child theme for the Pro theme. Treat it as the presentation layer of the site.

## Responsibility

- Put visual, layout, header, mobile navigation, cart/checkout presentation, and theme integration changes here.
- Do not put reusable WooCommerce business logic here when it can live in `meditrendy-core`.
- Do not edit the Pro parent theme, WordPress core, WooCommerce core, or third-party plugin files unless the user explicitly asks.

## Project Conventions

- Frontend/customer-facing labels should stay Lithuanian unless the user asks otherwise.
- Admin/editor-facing labels may be English or Polish.
- Prefer stable custom classes and existing theme modules over styling generated Cornerstone classes such as `m1q-*` or `e246-*`.
- Cornerstone/Pro generated markup can change; avoid depending on brittle generated class names when a custom wrapper or selector is possible.
- Keep changes focused. Avoid unrelated refactors.
- Always consider the user experience when making changes.
- Always consider website performance when making changes.
- Changes that apply to products usually apply to product sets as well.

## File Layout

- PHP bootstrap and enqueues: `functions.php`
- Theme modules: `modules/`
- CSS: `styles/`
- JavaScript: `scripts/`

Use existing files when the change naturally belongs there. Add a new file only when it creates a clearer boundary.

## JavaScript And CSS

- Use `const` and `let` in new JavaScript.
- Keep JavaScript small and scoped to the affected UI.
- Do not add new build tools or external packages without user approval.
- Avoid global selectors that could affect WooCommerce blocks, product pages, or Cornerstone sections unintentionally.
- Prefer CSS over JavaScript for styling.
- Prefer theme CSS for visual styling, even when the markup is generated by `meditrendy-core`.
- Keep plugin CSS to the minimum needed for plugin-owned structure, state, and fallback behavior.
- When a plugin feature needs broader visual polish, put the styling in this theme unless the CSS must ship with the shortcode/component to function.
- When writing JavaScript, use ES6 syntax and modern browser features.
- When writing JavaScript, avoid jQuery and other third-party libraries unless the existing integration requires them.
- When writing JavaScript, take performance into account.

## Testing

- For changed PHP files, run:
  `C:\laragon\bin\php\php-8.1.10-Win32-vs16-x64\php.exe -l path/to/file.php`
- For frontend behavior, verify the affected page in a browser when practical.
- Check both desktop and mobile behavior for navigation, header, filters, cart, and checkout UI changes.

## Boundaries

- If the request is about filters, waitlists, product queries, AJAX product reloads, subcategory shortcode logic, or WooCommerce data behavior, it probably belongs in `meditrendy-core`.
- If the request is about how something looks or behaves in the theme shell, it probably belongs here.
